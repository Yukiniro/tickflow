# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

Package manager is **pnpm** (pinned to `pnpm@11.5.0` via the `packageManager` field; pnpm 11 reads settings from [pnpm-workspace.yaml](pnpm-workspace.yaml), e.g. `allowBuilds`).

```bash
pnpm install        # install dependencies
pnpm dev            # start dev server at http://localhost:3000
pnpm build          # production build (Turbopack, the Next 16 default)
pnpm start          # serve the production build
pnpm lint           # eslint . (flat config, ESLint 9 + eslint-config-next 16)
pnpm format         # prettier --write over src/**/*.{ts,tsx,css}
```

There is **no test suite** in this repo — no test runner or test files are configured.

Note: Next 16 removed `next lint` and no longer runs ESLint during `next build`, so **lint never blocks `pnpm build`** — run `pnpm lint` explicitly to catch issues.

## Architecture

Next.js 16 (App Router) + React 19 + TypeScript, styled with Tailwind CSS v4. The app is a multi-style clock site; the import alias `@/*` maps to `src/*`.

### Internationalization (next-intl) — the routing backbone

Every page lives under the `[locale]` segment, so **all routes are locale-prefixed** (e.g. `/en/analog`, `/zh/digital`). The wiring spans several files:

- [src/i18n/routing.ts](src/i18n/routing.ts) — the single source of truth for `locales` (`en`, `zh`, `ja`) and `defaultLocale` (`en`).
- [src/proxy.ts](src/proxy.ts) — `next-intl` middleware that handles locale negotiation/redirects; its `matcher` excludes `api`, `_next`, `_vercel`, and any path with a dot. (Next 16 renamed the `middleware` file convention to `proxy`; the `next-intl/middleware` import is unchanged.)
- [src/i18n/request.ts](src/i18n/request.ts) — loads `src/i18n/locales/<locale>.json` as the message bundle per request.
- The `next-intl` plugin is applied in [next.config.ts](next.config.ts) via `createNextIntlPlugin()`.

**To add a locale:** add the code to `routing.ts`, create `src/i18n/locales/<code>.json`, and add the option in `src/components/language-switcher.tsx`. Forgetting any one of these breaks the locale silently.

### Clock types — the core feature

Six clock styles, each a route directory under `src/app/[locale]/` paired with a component in `src/components/clock/`:

| Route | Component |
|-------|-----------|
| `analog` | `analog-clock.tsx` |
| `basic` | `basic-clock.tsx` |
| `comic` | `comic-clock.tsx` |
| `digital` | `digital-watch.tsx` |
| `flip` | `flip-clock.tsx` / `flip-clock-client.tsx` |
| `led` | `led-clock.tsx` |

`clock-type-selector.tsx` and `nav.tsx` drive switching between them. **Adding a clock type** means: new route dir under `[locale]/`, new component in `clock/`, an entry in the selector, and i18n keys in each `locales/*.json`. (The [README.md](README.md) is out of date here — it only documents `basic/digital/comic/flip` and omits `analog` and `led`.)

### State — Jotai

Global state is Jotai atoms in [src/store/](src/store/) (`time.ts`, `sound.ts`, `background.ts`), with a root `<Provider>` in [src/app/providers.tsx](src/app/providers.tsx).

The clock heartbeat is centralized in [src/hooks/use-time.ts](src/hooks/use-time.ts): a **single `setInterval` ticks once per second**, writing `timeAtom`; `formattedTimeAtom` is a derived atom that handles 12/24-hour formatting. The same hook loads `/public/tick.mp3` through the Web Audio API and plays it each tick when `soundEnabledAtom` is on. Consume time via `useTime()` rather than adding new timers, so the app keeps one clock source.

### Backgrounds (Pexels)

[src/lib/pexels.ts](src/lib/pexels.ts) fetches background images from the Pexels API. It reads the key from **`NEXT_PUBLIC_PEXELS_API_KEY`** (client-exposed). [next.config.ts](next.config.ts) only whitelists `images.pexels.com` in `images.remotePatterns`, so background images must come from that host.

## Environment variables

Create `.env.local`:

```bash
NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX            # Google Analytics 4 (optional), wired via @next/third-parties
NEXT_PUBLIC_PEXELS_API_KEY=...            # Pexels API key for background images (optional)
```

Note: the README references `PEXELS_API_KEY`, but the code reads `NEXT_PUBLIC_PEXELS_API_KEY` — use the latter.

## Conventions

- Comments and commit messages in this repo are predominantly in Chinese; match the surrounding style.
- UI primitives in `src/components/ui/` are Radix UI + `class-variance-authority`; compose class names with the `cn()` helper (`clsx` + `tailwind-merge`) from `src/lib/utils/utils.ts`.
- Theme switching uses `next-themes`; SEO/PWA surfaces are generated by `src/app/{manifest,robots,sitemap}.ts` — update `sitemap.ts` when adding routes.
